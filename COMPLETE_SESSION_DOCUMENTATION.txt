9. UNDERSTANDING THE PIPELINE
==============================

HOW DATA FLOWS:

User Browser (localhost:5173)
  ↓ (frontend makes request via fetch/API)
Vite Dev Server (React)
  ↓ (proxies /api to backend)
FastAPI Backend (localhost:8000)
  ↓ (routes request)
Route Handler (routes/*.py)
  ↓ (calls service)
Service Layer (services/*.py)
  ↓ (calls pipeline)
Pipeline Processor (pipeline/*.py)
  ↓ (queries database)
SQLite Database (ocr_clinical.db)
  ↓ (returns data)
Response sent back to frontend
  ↓
Frontend displays results with Recharts visualizations


HOW OCR WORKS:

Input Image
  ↓
Segment into text lines (horizontal projection)
  ↓
For each line:
  - Convert to RGB PIL image
  - Run through TrOCR encoder (image → features)
  - TrOCR decoder generates text tokens
  - Decode tokens to text
  ↓
If TrOCR fails:
  - Fall back to Tesseract OCR
  ↓
Output text


HOW THE 4-STAGE PIPELINE WORKS:

Stage 1: OCR
- Input: Image or ground truth text
- If real mode: Run TrOCR on image
- If simulation mode: Inject errors at specified rate
- Output: OCR text with initial CER/WER metrics

Stage 2: Spell Correction
- Input: OCR text from Stage 1
- Process: Levenshtein-based spell correction
- Output: Corrected text with error recovery metrics

Stage 3: Summarization
- Input: Corrected text from Stage 2
- Process: Extractive summarization (keep important sentences)
- Output: Summary with ROUGE scores

Stage 4: Topic Classification
- Input: Text from Stage 2
- Process: Classify into medical topics
- Output: Predicted topic with confidence score

Metrics Tracked at Every Stage:
- CER (Character Error Rate) vs ground truth
- WER (Word Error Rate) vs ground truth
- ROUGE scores (for summaries)
- Processing time (milliseconds)
- Snowball factor (error amplification)


METRICS EXPLAINED:

CER (Character Error Rate):
- Measures character-level accuracy
- Lower is better (0% = perfect)
- Formula: (substitutions + deletions + insertions) / reference_length
- Example: If 5 characters wrong out of 100 = 5% CER

WER (Word Error Rate):
- Measures word-level accuracy
- Lower is better (0% = perfect)
- Same formula as CER but applied to words
- More lenient than CER

ROUGE Scores (for summarization):
- ROUGE-1: Unigram overlap with reference
- ROUGE-2: Bigram overlap with reference
- ROUGE-L: Longest common subsequence
- Higher is better (1.0 = perfect match)

Error Recovery Rate:
- How much spell correction improved the text
- Formula: (CER_before - CER_after) / CER_before × 100%
- Example: If CER went from 10% to 6% = 40% recovery

Snowball Factor:
- Error amplification through the pipeline
- If CER = 5% at OCR but 7% at end = factor of 1.4 (40% amplified)
- Shows how errors cascade through stages

12. TROUBLESHOOTING
===================

PROBLEM: Port 8000 Already in Use
SYMPTOMS: "Address already in use" when starting backend
SOLUTION:
  Option 1: Kill the other process
    - Run: lsof -i :8000 (on Mac/Linux)
    - On Windows: netstat -ano | findstr :8000
    - Kill: taskkill /PID [PID] /F
  Option 2: Use different port
    - Run: uvicorn app.main:app --port 8001
    - Update vite.config.js proxy to 8001

PROBLEM: C: Drive Still Full
SYMPTOMS: Installation fails due to disk space
SOLUTION:
  Step 1: Check space: df -h /c
  Step 2: Run Windows Disk Cleanup
  Step 3: Delete Downloads, old files
  Step 4: Restart installation

PROBLEM: TrOCR Model Won't Download
SYMPTOMS: "Timeout" or "Connection error" when running OCR
SOLUTION:
  Step 1: Check internet connection
  Step 2: Check C: drive has 5+ GB free
  Step 3: Check D: drive cache: ls .hf_cache/hub
  Step 4: Try again (may take 1-2 minutes for first use)

PROBLEM: Frontend Can't Connect to Backend
SYMPTOMS: "API error: 404" or "Cannot reach server"
SOLUTION:
  Step 1: Verify backend is running (see "Uvicorn running...")
  Step 2: Check vite.config.js proxy target is localhost:8000
  Step 3: Check frontend is on port 5173 (not 3000)
  Step 4: Check CORS allows localhost:5173

PROBLEM: Database is Locked
SYMPTOMS: "database is locked" error
SOLUTION:
  Step 1: Restart backend (Ctrl+C and restart)
  Step 2: If persists, delete and recreate: rm backend/ocr_clinical.db

PROBLEM: Module Not Found Error
SYMPTOMS: "No module named 'app'" or similar
SOLUTION:
  Step 1: Ensure backend/ is active directory: cd backend
  Step 2: Ensure venv is activated (should see (.venv) in prompt)
  Step 3: Reinstall: pip install -r requirements.txt

PROBLEM: React Component Won't Render
SYMPTOMS: Blank page or "Error loading page"
SOLUTION:
  Step 1: Check browser console (F12 Developer Tools)
  Step 2: Look for JavaScript errors
  Step 3: Check API requests (Network tab)
  Step 4: Restart frontend: Ctrl+C and npm run dev
